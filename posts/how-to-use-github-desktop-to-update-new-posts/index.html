如何使用 GitHub Desktop 来简化推送流程

需求>

需求 #

使用命令行指令来实现内容的发布与推送，操作“相对繁琐”且无直观可视的图形用户界面 (GUI)，对于「小白」而言，门槛相对较高。而 GitHub Desktop 具有简洁的 GUI，相较于命令行指令，操作简便且直观可视。配置流程一劳永逸。

使用方法>

使用方法 #

  1. 从 GitHub Desktop 下载相应操作系统的软件并安装。
  2. 按照 如何用 GitHub Pages + Hugo 搭建个人博客 中的操作流程，将本地博客文件夹下的 public 文件夹初始化为 Git 仓库，并通过 SSH 链接，将本地 public 文件夹关联至远端 GitHub Pages 仓库。
  3. 打开 GitHub Desktop 并将本地 public 文件夹作为其工作目录。
  4. 抓取远端 GitHub Pages 仓库信息并与本地 public 文件夹同步。
  5. 填写更新日志并将本地更新后的文件推送至远端 GitHub Pages 仓库主分支。
  6. 等待推送完成。
  7. 以后每次使用，先按照第 4 步，抓取远端信息与本地同步，再将本地文件推送即可。
